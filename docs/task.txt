Перед тем, как пригласить Вас на собеседование, предлагаем Вам выполнить небольшое ТЗ для более предметного диалога на собеседовании:

Напишите, средствами PHP, MySQL, JavaScript форму входа/регистрации нового пользователя. В результате заполнения формы пользователь должен предоставить информацию о себе (имя, логин, email, фото). После входа должен отображаться профайл зарегистрировавшегося пользователя.

Функционал:
1. Все формы должны быть выполнены способом, понятным пользователю, содержать необходимые инструкции, комментарии и т.п. (usability);
2. Кроме введения текстовых данных пользователь при регистрации должен иметь возможность загрузить графический файл форматов gif, jpg, png;
3. Должна быть возможность переключения языка интерфейса формы на другой язык;
4. Формы должны содержать как проверки так и вспомогательные тексты (названия полей, подсказки, ошибки и пр.);
5. Страницы должны иметь эстетический внешний вид.

Требования:
1. Код должен быть написан без использования фреймворков;
2. Проект должен содержать Docker файл и уметь запускать приложение в контейнере;
3. Код должен быть написан понятно и аккуратно, с соблюдением табуляции и прочих элементов написания, без лишних элементов и функций, не имеющих отношения к функционалу тестового задания, снабжен понятными комментариями;
4. Скрипт должен содержать средства верификации и валидации полей, а также защиту от некорректного ввода данных, спецсимволов, попыток взлома и т.п;
5. Валидация и верификация полей должна проводиться как на клиентской стороне, так и на серверной стороне (средствами PHP);
6. Структура базы данных должна быть обоснованной;
7. К проекту должен прилагаться текстовый файл с инструкцией по развертыванию и запуску.

Отсутствие или нарушение данных пунктов вероятнее всего послужит причиной отказа.

Временем и инструментом для выполнение тестового задания Вы не ограничены. Любые другие аспекты реализации, которые не указаны в требованиях, могут быть выполнены на Ваше усмотрение
